## 1. Overview

This document describes the API interface between the Rust core engine and the Android Kotlin UI. The interface is defined using UniFFI (Universal Foreign Function Interface) and supports async operations via Tokio.

### 1.1 Architecture

```
┌─────────────────┐         ┌──────────────────┐
│   Kotlin UI     │         │   Rust Engine    │
│   (Android)     │◄───────►│   (Native)       │
│                 │  UniFFI │                  │
│ - ViewModels    │  JNI    │ - BookingAgent   │
│ - Repositories  │  Bridge │ - HTTP Client    │
│ - UI State      │         │ - Scrapers       │
└─────────────────┘         └──────────────────┘
```

## 2. Core Interface

### 2.1 BookingAgent

Main entry point for all booking operations.

#### Constructor

```kotlin
// Kotlin usage
```

```rust

impl BookingAgent {
```

**Thread Safety:** Thread-safe, can be shared across coroutines.

#### getStatus()

Retrieves current agent status.

```kotlin
// Kotlin signature (generated by UniFFI)

```

```rust
// Rust signature

```

**Returns:** `EngineStatus` - Current operational state

**Example:**
```kotlin

    val status = agent.getStatus()
    uiState = uiState.copy(
        isRunning = status.isRunning,
        lastLog = status.lastLog,
        currentRunId = status.currentRunId
    )
}
```

#### startStrike()

Initiates a booking strike operation.

```kotlin
// Kotlin signature
suspend fun startStrike(configJson: String, runId: String): Boolean
```

```rust
// Rust signature
#[uniffi::export]
impl BookingAgent {
    pub async fn start_strike(&self, config_json: String, run_id: String) 
        -> Result<bool, EngineError>
}
```

**Parameters:**

| Parameter | Type | Description |
|-----------|------|-------------|
| `configJson` | `String` | JSON-serialized AppConfig |
| `runId` | `String` | Unique identifier for this run |

**Returns:** 
- `true` - Booking successful or alert triggered
- `false` - Strike completed without success
- Throws `EngineError` on failure

**Example:**
```kotlin
lifecycleScope.launch {
    try {
        val configJson = Json.encodeToString(appConfig)
        val success = agent.startStrike(configJson, runId)

        if (success) {
            showSuccess("Booking confirmed!")
        } else {
            showInfo("No availability found")
        }
    } catch (e: EngineError.ConfigError) {
        showError("Invalid configuration")
    } catch (e: EngineError.NetworkError) {
        showError("Network connection failed")
    } catch (e: EngineError.BookingFailed) {
        showError("Booking attempt failed")
    }
}
```

**Execution Flow:**
1. Parse configuration JSON
2. Validate credentials and site config
3. Wait for pre-warm time (if configured)
4. Execute precision timing loop
5. Monitor availability
6. Attempt booking (if mode = booking)
7. Return result

**Cancellation:**
Use `stop()` to cancel an in-progress strike.

---

#### stop()

Cancels the current operation.

```kotlin
// Kotlin signature
fun stop()
```

```rust
// Rust signature
#[uniffi::export]
impl BookingAgent {
    pub fn stop(&self)
}
```

**Description:** Sends cancellation signal to running strike. Operation will terminate at next await point.

**Thread Safety:** Safe to call from any thread.

**Example:**
```kotlin
Button(onClick = { agent.stop() }) {
    Text("Cancel")
}
```

### 3.2 EngineError

```kotlin
// Kotlin sealed class (generated)
sealed class EngineError : Exception() {
    class ConfigError : EngineError()
    class NetworkError : EngineError()
    class BookingFailed : EngineError()
    class InternalError : EngineError()
}
```

```rust
// Rust enum
#[derive(Debug, thiserror::Error, uniffi::Error)]
pub enum EngineError {
    #[error("Configuration error")]
    ConfigError,
    #[error("Network failure")]
    NetworkError,
    #[error("Booking failed")]
    BookingFailed,
    #[error("Internal engine error")]
    InternalError,
}
```

**Error Types:**

| Error | Description | Recovery |
|-------|-------------|----------|
| `ConfigError` | Invalid JSON or missing fields | Validate config before sending |
| `NetworkError` | Connection timeout or DNS failure | Retry with backoff |
| `BookingFailed` | Server error or slot taken | Alert user, suggest retry |
| `InternalError` | Unexpected Rust panic | Report bug, restart app |

## 8. Kotlin Integration Examples

### 8.1 Basic Usage

```kotlin
class BookingViewModel : ViewModel() {
    private val agent = BookingAgent()
    private val _uiState = MutableStateFlow(AgentUiState())
    val uiState: StateFlow<AgentUiState> = _uiState.asStateFlow()

    fun startBooking(config: AppConfig, runId: String) {
        viewModelScope.launch {
            _uiState.update { it.copy(isLoading = true) }

            try {
                val configJson = Json.encodeToString(config)
                val success = agent.startStrike(configJson, runId)

                _uiState.update { 
                    it.copy(
                        isLoading = false,
                        isSuccess = success,
                        message = if (success) "Booked!" else "No slots"
                    )
                }
            } catch (e: EngineError) {
                _uiState.update {
                    it.copy(
                        isLoading = false,
                        error = e.toString()
                    )
                }
            }
        }
    }

    fun cancel() {
        agent.stop()
    }

    override fun onCleared() {
        super.onCleared()
        agent.stop()
    }
}
```

### 8.2 Status Monitoring

```kotlin
@Composable
fun StatusMonitor(agent: BookingAgent) {
    var status by remember { mutableStateOf<EngineStatus?>(null) }

    LaunchedEffect(Unit) {
        while (isActive) {
            status = agent.getStatus()
            delay(100) // Poll every 100ms
        }
    }

    StatusCard(status = status)
}
```

### 8.3 Error Handling

```kotlin
fun handleError(error: EngineError): String {
    return when (error) {
        is EngineError.ConfigError -> 
            "Configuration error. Check your settings."
        is EngineError.NetworkError -> 
            "Network connection failed. Check internet."
        is EngineError.BookingFailed -> 
            "Booking failed. Slot may be taken."
        is EngineError.InternalError -> 
            "Internal error. Please restart app."
    }
}
```

## 9. Best Practices

### 9.1 Configuration Validation

Always validate configuration before calling `startStrike()`:

```kotlin
fun validateConfig(config: AppConfig): Boolean {
    return config.sites.isNotEmpty() &&
           config.scheduled_runs.isNotEmpty() &&
           config.credentials.isNotEmpty()
}
```

### 9.2 Resource Management

Always stop the agent when done:

```kotlin
DisposableEffect(Unit) {
    onDispose {
        agent.stop()
    }
}
```

### 9.3 Thread Safety

The `BookingAgent` is thread-safe, but prefer single coroutine for operations:

```kotlin
// Good
viewModelScope.launch {
    agent.startStrike(config, id)
}

// Avoid - concurrent calls
launch { agent.startStrike(config1, id1) }
launch { agent.startStrike(config2, id2) }
```
